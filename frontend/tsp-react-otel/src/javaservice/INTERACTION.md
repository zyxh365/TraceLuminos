## 交互流程

### 场景1：查看所有Trace（默认模式）

```
步骤1：打开页面
        ↓
步骤2：选择时间范围（可选）
        ↓
步骤3：点击"刷新"按钮
        ↓
步骤4：右侧显示Trace列表
        ↓
步骤5：左侧显示聚合拓扑图
        ↓
步骤6：点击某条Trace
        ↓
步骤7：右侧显示Trace详情
        左侧显示单次Trace拓扑图
```

**效果**：
- 右侧面板显示标题："Trace 列表 (共100条)"
- 每条Trace显示：TraceID、状态、耗时、操作名称、时间
- 左侧拓扑图边显示："调用次数 · 平均耗时"

---

### 场景2：搜索TraceID

```
步骤1：在搜索框输入TraceID（如 "abc"）
        ↓
步骤2：点击搜索按钮（或按回车）
        ↓
步骤3A：如果只搜索到1条结果
        → 右侧显示"Trace 详情"
        → 左侧显示该Trace的单次拓扑图
        → 拓扑图边显示："响应耗时"

步骤3B：如果搜索到多条结果
        → 右侧显示"Trace 列表 (共N条)"
        → 左侧显示聚合拓扑图
        → 用户点击某条Trace查看详情
```

**效果**：
- 搜索到1条：直接进入详情模式
- 搜索到多条：显示列表，让用户选择

---

### 场景3：返回列表模式

```
当前在详情模式
        ↓
点击右侧详情面板的 × 按钮
        ↓
返回列表模式
        ↓
- 右侧恢复显示Trace列表
- 左侧恢复显示聚合拓扑图
- 当前选中的Trace取消选中
```

## 右侧面板行为

### Trace列表区域

**显示内容**：
- 标题栏：显示 "Trace 列表" 和总数
- 列表项：每条Trace一行
- 空状态：提示 "点击刷新按钮加载Trace列表"

**列表项显示**：
- TraceID（前14位 + ...）
- 状态图标（✓ OK / ✗ ERR）
- 根Span耗时
- 操作名称
- 时间戳

**交互**：
- 鼠标悬停：背景变深色
- 点击：选中该Trace
  - 选中项：紫色左边框 + 背景
  - 右侧展开详情面板
  - 左侧切换到单次拓扑图

### Trace详情区域

**显示条件**：
- 选中某条Trace后显示
- 未选中时隐藏

**显示内容**：
- 标题栏：显示 "Span 详情"
- TraceID（前14位 + ...）
- 整体状态（✓ / ✗）
- 总耗时
- Span树形结构

**Span树显示**：
- 服务名称（带颜色）
- 操作名称
- 耗时（带进度条）
- Tags（前5个）
- 调用层级（缩进显示）

**关闭详情**：
- 点击标题栏的 × 按钮
- 返回列表模式
- 保持Trace列表选中状态

## 拓扑图行为

### 全局聚合模式

**节点**：
- 显示所有服务
- 大小一致
- 可拖拽

**边**：
- 显示格式："调用次数 · 平均耗时"
- 例如："350 · 5ms"
- 箭头指向被调用方

### 单次Trace模式

**节点**：
- 只显示该Trace涉及的服务
- 大小一致
- 可拖拽

**边**：
- 显示格式："响应耗时"
- 例如："5ms"
- 箭头指向被调用方

## 配置选项

### 合并网关

勾选后：
- 将 `kong-gateway:/api/*` 等端点节点合并为 `kong-gateway`
- 减少节点数量，避免拓扑图混乱

### 时间范围

可选范围：
- 30分钟
- 1小时（默认）
- 2小时
- 6小时
- 24小时

切换时间范围后：
- 自动重新加载Trace列表
- 重新生成聚合拓扑图
- 保持当前选中的Trace（如果还在时间范围内）

---

## 错误处理

### 无数据情况

```
该时间段内暂无链路数据
```
- 显示在状态提示区域（黄色警告）
- 拓扑图显示：暂无数据
- 右侧面板显示：点击刷新按钮加载Trace列表

### 搜索无结果

```
未找到TraceID包含 "xxx" 的链路
```
- 显示在错误提示区域（红色错误）
- 右侧面板清空
- 拓扑图显示：暂无数据

### 接口错误

```
加载失败: 网络错误
```
- 显示在错误提示区域（红色错误）
- 保持上一个有效状态

## 性能优化

### 数据量限制
- 默认加载：100条Trace
- 搜索结果：50条Trace
- 单次Trace的Span数：无限制（建议后端也限制）

### 拓扑图优化
- 节点数量限制：建议不超过50个
- 边数量限制：建议不超过100条
- 物理模拟：600帧后停止

## 与原TopologyView的差异

| 功能 | 原TopologyView | JavaServiceTopology |
|------|---------------|---------------------|
| 数据源 | Jaeger / 华为云APM | Java服务 |
| 默认行为 | 显示服务列表 | 显示Trace列表 |
| 搜索方式 | TraceID / VIN码 | 仅TraceID |
| 右侧面板 | Trace列表 + 详情 | Trace列表 + 详情（相同） |
| 拓扑图 | 服务视图 / 端点视图 | 聚合视图 / 单次视图 |
| 网关合并 | 支持 | 支持 |
| 数据格式 | Jaeger格式 | Jaeger格式（相同） |
