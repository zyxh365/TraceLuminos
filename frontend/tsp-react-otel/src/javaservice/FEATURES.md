## 主要功能

### 1. 默认模式（聚合拓扑图）

**触发条件**：
- 页面初始加载
- 点击刷新按钮
- 切换时间范围
- 从搜索模式返回

**展示内容**：
- 拓扑图显示所有服务节点
- 边显示格式：`调用次数 · 平均耗时`
- 例如：`350 · 5ms` 表示调用了350次，平均耗时5毫秒

**数据来源**：
```
GET /traces?startTime={timestamp}&endTime={timestamp}&limit=100
```

---

### 2. 搜索模式（TraceID搜索）

**触发条件**：
- 在搜索框输入TraceID并点击搜索

**展示内容**：
- 如果搜索到多条结果：显示在右侧面板的Trace列表
- 如果只有1条结果：直接显示该Trace的拓扑图
- 拓扑图边显示：`响应耗时`（单次调用）

**数据来源**：
```
GET /traces/search?traceId={traceId}&startTime={timestamp}&endTime={timestamp}
```

---

### 3. 右侧面板

#### 3.1 Trace列表

**显示时机**：
- 搜索到多条Trace时

**功能**：
- 列出所有搜索到的Trace
- 每条Trace显示：
  - TraceID（前14位）
  - 状态（✓ OK / ✗ ERR）
  - 根Span耗时
  - 操作名称
  - 时间戳

**交互**：
- 点击Trace项查看详情

#### 3.2 Trace详情

**显示时机**：
- 选中某条Trace后

**功能**：
- 显示TraceID
- 显示整体状态和耗时
- 以树形结构展示所有Span
- 每个Span显示：
  - 服务名称（带颜色）
  - 操作名称
  - 耗时（带进度条）
  - Tags（前5个）
  - 错误状态

## 交互说明

### 搜索TraceID

1. 在搜索框输入TraceID（支持模糊匹配）
2. 按回车或点击搜索按钮
3. 右侧面板显示搜索结果
4. 点击某条结果查看详情

### 查看单次Trace

1. 搜索或选择Trace后，拓扑图切换到单次模式
2. 显示该Trace的调用链路
3. 边显示单次调用的响应耗时
4. 右侧显示Span树形详情

### 返回全局模式

1. 点击右侧Trace详情面板的 × 按钮
2. 或点击搜索框旁的 ✕ 按钮
3. 拓扑图恢复全局聚合模式

---

## 配置选项

### 合并网关

勾选"合并网关"后：
- 将 `kong-gateway:/api/*` 等端点节点合并为 `kong-gateway`
- 避免网关节点过多导致拓扑图混乱

### 时间范围

支持选择：
- 30分钟
- 1小时（默认）
- 2小时
- 6小时
- 24小时

调整时间范围后自动刷新拓扑图。
